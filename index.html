    Category One

    Software Design & Engineering

    The artifact used is an Inventory app developed using Android Studio, Java, and SQLite. It was created in the CS 360 Mobile Architecture and Programming class. The purpose of the app is to allow users to store products and their respective quantities in a database to manage inventory. I am converting the app into a Biological Sample Inventory System. This app will store biological samples in a database and include extra features such as a search function, data visualization, and exporting data into files. This artifact was chosen because although the final project was completed in the class, there was room for improvement. An inventory app can be modified for many purposes, and in the field of bioinformatics, gathering samples in a secure and efficient manner is useful for researchers and other scientists.

    The architecture seen in the enhancement is the Model-View-ViewModel (MVVM) pattern, which improves the application's design by separating the UI from the program's logic. The folder structure was reorganized to improve the organization of files, including the activities, as well as the adapter, utils, and viewholder packages, which are part of the RecyclerView list implementation. The addition of search functionality and allowing users to select items from the list is done through the RecyclerView structure. Other features added to enhance the application include data visualization via a pie chart and the ability to export data in CSV format.

    The enhancements completed thus far cover all course outcomes. Key areas addressed include database implementation, computational solutions through the use of different algorithms and tools, and structural adjustments, such as using RecyclerView and LiveData, which enhance the design and resolve technical issues. Additionally, algorithms have been improved for gathering and storing data, such as filtering and searching for specific samples. Security and role-based access are implemented through proper registration mechanisms. Adding a search feature will be very useful for users when looking for a sample in a large dataset. The export to CSV feature could also be beneficial for research purposes in the bioinformatics field. Modifications to the app's build and dependencies were necessary to incorporate pie chart visualization into the application.

    Overall, the process has been educational, informative, and somewhat time-consuming. Some new skills that had to be researched included exporting to CSV files, search implementation, and importing a new dependency to visualize data with a pie chart. In addition to these new techniques, new components were added to the application, such as a dropdown menu, a menu bar, and a calendar selection. Many new features and skills were incorporated, and I found that much more research was needed than I had originally anticipated. Time was also spent debugging some newly added features, which was challenging. For example, the search feature was initially difficult because I wanted users to be able to search by sample name, type, or species. This required adjusting the query for filtering to include all fields and iterating over all data for samples that match the search criteria into a LiveData array. Achieving the desired functionality involved research and multiple iterations; however, the functionality can now be viewed in the Database Helper. Additionally, warnings from the original artifact were addressed, and the file structure was adjusted to further enhance the application.

    Category Two

    Algorithms & Data Structures

    The artifact used for category two is a project from class CS 360 Mobile Architect and Programming. It is an inventory application to store products for tracking purposes and it utilizes Android Studio, Java and SQLite. I converted this mobile app into a Biological Samples Inventory System. This app will store biological samples into a database and offer different features to enhance user experience and aid in analysis. This artifact has been chosen due to the quality of work previously submitted and room for improvement. The goal of tracking and organizing data could be applied to many different fields and the project will aim for the bioinformatics field. Certain skills could be displayed for this category such as efficient search algorithm and choice of data structure to create the list of items. Other features such as a pie chart have also been completed which required an outside library and algorithms to properly load data into the chart.

    The DatabaseHelper displays the core functionality of the app in terms of displaying and offering functionality related to samples. A few new methods that have been added to accomplish these enhancements were the loadSamples() method to ensure the list of samples is visible, searchSamples() function will return a LiveData Array List of samples, getSampleByID() which aids the selection of a sample to view more details, and getSamplesTypeCounts() which utilizes a Map to store the type and count of samples for the pie chart. The use of DiffUtil and ViewHolder had to be included since RecyclerView was chosen to contain the list of items, and these files ensure the list of samples is updated appropriately and the individual sample item is clickable. The previous choice of structure was the GridView layout which displayed the products from the inventory in a table like structure, however this enhancement makes the list much more user friendly and organized, with added features for better user experience.

    The outcomes that this enhancement aligns with is using algorithmic principles and utilizing different techniques & tools to display computer solutions. The addition of a search feature, clickable samples to view details, and a pie chart for data analysis have all been new additions to the original artifact. The design has been enhanced, and through the use of algorithms and different structures, the application is more useful for the user for the purpose of tracking samples in an app.

    This enhancement was educational and required trial and error to settle with structure decisions and ensure the algorithms are accomplishing the expected requirement. During this enhancement there were a few challenges that I faced. There were different library options for displaying data through a pie chart. I had to experiment to find which library is most useful and easy to integrate into the app, and the choice I went with is MPAndroidChart. Once I created the function to gather sample types and allow the activity to have access to this data, the issue I found was the use of the LiveData for the pie chart. After going over documentation I understood that this prevented the pie chart from properly loading data into the structure. I was able to adjust the pie chart activity & methods, while also adding customizations to display the data in the chart accordingly.

    Category Three

    Databases

    The artifact used for this category is the inventory app from CS 360 Mobile Architecture & Programming class. It utilizes Android Studio, Java and SQLite. The purpose of the application is to store inventory and display all products in an organized manner. There are CRUD operations seen in the app, where on the dashboard page a user could edit the quantity of the product as well as delete the product from the database. The user could read all products, and they could also add products as needed. The application includes a basic login & register functionality. The enhancement is converting this basic app to a Biological Samples Inventory System, where the goal of storing items for users is still the same, but the designs and functionality will be updated. This was done to show that the functionality of tracking items could be used in many different fields, and this app will display new skills and techniques to enhance user functionality and security.

    The first feature added was to update the database to switch from products to biological samples. This meant to add more fields, as well as implement user roles during registration. Another new feature is hashing passwords to enhance security through the algorithm SHA-256 as well as implementing try-catch blocks where necessary. All CRUD operations are visible in the app, but rather than these operations embedded within the list of items, this was moved to a different screen where users could edit or delete as needed. This allows the dashboard page to be more user friendly and organized. A new feature has been added which is to export samples as a CSV file which is commonly utilized in the bioinformatics field.

    The course outcomes seen in this category enhancements are mainly four, which are building collaborative environments for diverse audiences, design and develop work, utilizing tools to produce computer solutions, and also enhancing security to mitigate concerns and vulnerabilities. Overall, the process of making additions to the database category was successful. I learned how to add roles to an application, as well as implementing different data types such as Enum for storing a list of predefined values and how to display this visually for the users. Also, the exporting files functionality has never been attempted in previous projects so this was a little challenging. After conducting research with some trial and error the functionality was completed successfully. I learned the general steps on how to write to files by gathering data, appending the information using the FileWriter, then flushing and closing the stream.
